# Інструкція з генерації документації

## Загальна інформація

У проєкті для автоматичної генерації документації використовується **JSDoc**.

Документація генерується окремо для двох частин системи:

- `my-app` — frontend-частина застосунку;
- `server` — backend-частина застосунку.

Згенерована документація зберігається в таких папках:

```text
my-app/docs/generated/frontend
server/docs/generated/backend
```

---

## Передумови

Перед генерацією документації необхідно переконатися, що:

- у проєкті встановлено всі залежності;
- у файлах коду додані JSDoc-коментарі для публічних функцій, компонентів, middleware, маршрутів API та допоміжних модулів;
- у папках `my-app` і `server` налаштовані файли `jsdoc.json`;
- у `package.json` для обох частин проєкту додано скрипт:

```json
"docs": "jsdoc -c jsdoc.json"
```

---

## Генерація документації для frontend

Перейдіть у папку `my-app`:

```bash
cd my-app
```

Встановіть залежності:

```bash
npm install
```

Згенеруйте документацію:

```bash
npm run docs
```

Після виконання команди HTML-документація буде створена у папці:

```text
my-app/docs/generated/frontend
```

---

## Генерація документації для backend

Перейдіть у папку `server`:

```bash
cd server
```

Встановіть залежності:

```bash
npm install
```

Згенеруйте документацію:

```bash
npm run docs
```

Після виконання команди HTML-документація буде створена у папці:

```text
server/docs/generated/backend
```

---

## Що потрібно документувати перед генерацією

Перед запуском генерації документації необхідно переконатися, що JSDoc-коментарі додані для:

- публічних функцій;
- React-компонентів;
- Redux thunk-ів, reducers і slice-ів;
- middleware;
- маршрутів API;
- утиліт для роботи з токенами і файловим сховищем;
- важливої бізнес-логіки;
- складних або нетривіальних алгоритмів;
- взаємодії між компонентами системи.

---

## Перевірка результату

Після генерації документації потрібно:

1. перейти в папку `my-app/docs/generated/frontend` або `server/docs/generated/backend`;
2. знайти файл `index.html`;
3. відкрити його у браузері;
4. перевірити, чи коректно відображаються модулі, функції, параметри та описи.

---

## Оновлення документації

Після будь-яких змін у публічних інтерфейсах коду потрібно:

1. оновити JSDoc-коментарі;
2. повторно виконати генерацію документації;
3. перевірити результат;
4. за потреби оновити `README.md` та інші файли документації.

---

## Архівація документації для здачі

Після генерації документації необхідно заархівувати згенеровані папки з документацією.

Наприклад, можна створити архів:

```text
generated_docs.zip
```

До архіву потрібно включити:

```text
my-app/docs/generated/frontend
server/docs/generated/backend
```

Цей архів додається до лабораторної роботи як окремий файл із згенерованою документацією.
